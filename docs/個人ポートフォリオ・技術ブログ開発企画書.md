# **個人ポートフォリオ・技術ブログ開発企画書**

## **1\. プロジェクト概要**

### **プロジェクト名**

Personal CG & Research Lab (仮)

### **コンセプト**

「創作と研究の融合」  
WebGL/WebGPUを用いたCG作品、基礎技術の解説、および研究成果を集約するポートフォリオサイト。単なる情報の羅列ではなく、閲覧者がブラウザ上で実際に技術に触れられる「体験型」のコンテンツを提供する。

### **ターゲット読者**

* CG技術、レンダリング技術に興味があるエンジニア・学生  
* 研究成果に関心を持つ研究者  
* 採用担当者（技術力と実装力の証明）

## **2\. サイト構成（Sitemap）**

1. **Home (トップページ)**  
   * **FV (First View):** 全画面WebGLデモ（Stable Fluidsやパーティクル表現など、視覚的インパクト重視）。  
   * **About:** 簡潔な自己紹介とサイトの目的。  
   * **Recent Posts:** 最新のブログ記事や作品へのリンク。  
2. **Profile (プロフィール)**  
   * 経歴、スキルセット、使用ツール。  
   * SNSリンク (GitHub, X, etc.)。  
3. **Research (研究業績)**  
   * 論文リスト、学会発表履歴。  
   * 各研究の要約とPDFへのリンク、補足資料。  
4. **Portfolio (CG作品集)**  
   * **一覧ページ:** カテゴリ別（Simulation, Raymarching, WebGPUなど）のサムネイルグリッド。  
   * **詳細ページ:**  
     * 全画面表示切り替え可能なCanvasエリア。  
     * 技術解説、パラメータ調整UI（Levaなどを利用）。  
5. **Blog (技術ブログ/日記)**  
   * **技術記事:** 記事本文中にReactコンポーネントとしてWebGL Canvasを埋め込み、読みながら挙動を確認できる構成（MDXを利用予定）。  
   * **日記:** 研究の進捗や日々の雑記。  
   * **機能:** タグ機能、カテゴリ分け、検索機能。  
6. **Common (共通部分)**  
   * **Header:** ナビゲーション。透過処理などでCanvasの邪魔をしないデザイン。  
   * **Footer:** コピーライト、主要リンク。

## **3\. デザイン・UI/UX要件**

* **没入感:** Three.js (React Three Fiber) を活用し、Webサイト全体を一つのアプリケーションのように感じさせる。  
* **レスポンシブ:** PCではリッチな3D表現、モバイルではパフォーマンスを考慮した軽量な表示またはレイアウトの最適化を行う。  
* **ダークモード:** CG作品が映えるよう、ベースはダークテーマまたは黒基調とする。  
* **インタラクション:** 記事内の数式やコードと、3Dモデルの動きが連動するような仕組みを目指す。

## **4\. 技術スタック解説**

初心者の学習コストと、将来的な拡張性を天秤にかけ、以下の構成を採用する。

| 領域 | 採用技術 | 選定理由 |
| :---- | :---- | :---- |
| **Runtime** | **Node.js** | JS実行環境のデファクトスタンダード。Next.jsの動作に必須。 |
| **Framework** | **Next.js (App Router)** | Reactのフレームワーク。ページ遷移が高速で、ブログ構築（SSG/SSR）に最適。Vercelとの相性が抜群。 |
| **UI Library** | **React** | コンポーネント指向でUIを作るライブラリ。宣言的にUIを書けるため管理が楽。 |
| **3D Library** | **Three.js** | WebGLを扱いやすくするライブラリ。 |
| **Integration** | **React Three Fiber (R3F)** | Reactの中でThree.jsを「宣言的」に書けるライブラリ。Reactのstate管理と3Dシーンを連動させるのに必須。 |
| **Styling** | **Tailwind CSS** | クラス名を書くだけでスタイルが当たるCSSフレームワーク。CSSファイルを行ったり来たりしなくて済むため開発速度が上がる。 |
| **Contents** | **MDX** | Markdownの中にReactコンポーネント（3Dキャンバス）を直接書けるフォーマット。技術ブログの核となる。 |
| **AI Tool** | **GitHub Copilot** | コード補完AI。未経験の構文やボイラープレート（定型文）の記述を強力にサポート。 |
| **Deploy** | **Vercel** | Next.jsの開発元が提供するホスティングサービス。GitHubにプッシュするだけで自動公開される。 |

## **5\. 開発ロードマップ（初期フェーズ）**

1. **環境構築:** Node.js, VSCode, GitHub連携。  
2. **モックアップ作成:** Hello WorldレベルのNext.jsアプリを立ち上げ、Vercelへデプロイしてみる。  
3. **3D導入:** React Three Fiberをインストールし、トップページに立方体を回す。  
4. **骨組み作成:** ヘッダー、フッター、ページ遷移の実装。  
5. **ブログ機能実装:** MDXのセットアップ、記事投稿フローの確立。  
6. **ポートフォリオ実装:** 過去のWebGLコードをReactコンポーネント化して移植。  
7. **デザイン調整:** Tailwind CSSで見た目を整える。
