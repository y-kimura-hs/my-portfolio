# **WebGLポートフォリオサイト 実装ロードマップ**

## **フェーズ 1: サイトの骨組みを作る (所要目安: 1〜2日)**

まずは3D以外の「普通のWebサイト」としての外枠を作ります。これにより、開発中に迷子になりにくくなります。

* \[ \] **1-1. 不要なファイルの掃除**  
  * src/app/page.tsx や src/app/globals.css の初期コードを整理し、真っ白なキャンバスにする。  
* \[ \] **1-2. フォルダ構成の作成**  
  * src の中に以下のフォルダを作成して整理整頓の準備をする。  
    * components (ボタンやヘッダーなどの部品)  
    * components/canvas (3D関連の部品)  
    * components/dom (HTML関連の部品)  
* \[ \] **1-3. ヘッダーとフッターの実装**  
  * src/components/dom/Header.tsx などを作成。  
  * どのページに行っても表示されるように src/app/layout.tsx に配置する。  
  * **ゴール:** 画面の上部にタイトル、下部にコピーライトが表示されている状態。  
* \[ \] **1-4. ページ遷移の作成**  
  * Next.jsの仕組みを使って空のページを作る。  
  * src/app/about/page.tsx (自己紹介)  
  * src/app/blog/page.tsx (ブログ一覧)  
  * src/app/works/page.tsx (作品一覧)  
  * **ゴール:** ヘッダーのリンクをクリックすると、URLが切り替わりページが遷移する状態。

## **フェーズ 2: WebGL (React Three Fiber) の導入 (所要目安: 2〜3日)**

あなたの強みであるCGを表示します。ここが一番楽しいパートです。

* \[ \] **2-1. Canvasコンポーネントの作成**  
  * src/components/canvas/Scene.tsx を作成し、\<Canvas\> タグを配置。  
  * これをトップページ (src/app/page.tsx) に配置。  
* \[ \] **2-2. 基本的な3Dオブジェクトの表示**  
  * 立方体 (\<boxGeometry\>) を出し、useFrame を使って回転させる。  
  * **ゴール:** 黒い背景の中に、回転する立方体が表示されている。  
* \[ \] **2-3. シェーダーのテスト (GLSL)**  
  * shaderMaterial を使って、自作のフラグメントシェーダーを適用してみる。  
  * React内でのシェーダーの書き方（glsl-react や 文字列としての記述）に慣れる。  
* \[ \] **2-4. 全画面背景の実装**  
  * トップページ背景に、Stable Fluidsやパーティクルなどのデモを表示する。  
  * z-index (重なり順) を調整し、文字の後ろに3Dがある状態を作る。

## **フェーズ 3: ブログ・記事機能の実装 (所要目安: 3〜4日)**

「記事の中でデモが動く」機能を作ります。少し難易度が上がりますが、ここがこのサイトの目玉です。

* \[ \] **3-1. MDXの導入**  
  * Markdownの中でReactコンポーネントを使えるようにするライブラリ (@next/mdx や contentlayer など) をセットアップ。  
* \[ \] **3-2. 記事データの作成**  
  * posts フォルダを作り、テスト記事 hello-world.mdx を置く。  
* \[ \] **3-3. 記事内への3D埋め込みテスト**  
  * MDXファイルの中に \<My3DComponent /\> と書いて、記事の途中でCanvasが動くか確認する。  
  * **ゴール:** 文章の合間に、インタラクティブな3Dデモが表示されている。

## **フェーズ 4: ポートフォリオと仕上げ (所要目安: 2〜3日)**

作品集として体裁を整えます。

* \[ \] **4-1. 作品データの定義**  
  * 作品のタイトル、画像、説明などをまとめたデータファイルを作成。  
* \[ \] **4-2. 作品一覧ページ (Grid表示) の作成**  
  * Tailwind CSSのGrid機能を使って、サムネイルを綺麗に並べる。  
* \[ \] **4-3. デザインの調整**  
  * フォントの調整、余白の調整。  
  * レスポンシブ対応（スマホで見た時に崩れていないか確認）。

## **フェーズ 5: 公開 (所要目安: 1時間)**

世界に公開します。

* \[ \] **5-1. GitHubへプッシュ**  
  * 書いたコードをGitHubにアップロード。  
* \[ \] **5-2. Vercelと連携**  
  * Vercelの画面でGitHubリポジトリを選択してデプロイボタンを押す。  
  * **ゴール:** あなただけのURL (https://your-name.vercel.app) で誰でもアクセスできるようになる。
